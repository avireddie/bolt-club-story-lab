# Generate story

Use this prompt when asking an assistant to draft or expand a user story from requirements.

## Instructions for the model

- Produce a story that follows `story-template.md` (summary, goals, non-goals, acceptance criteria).
- Keep acceptance criteria testable and unambiguous.
- Call out dependencies, risks, and open questions explicitly.

## User fill-in

Describe the feature or problem below, then paste this block into your assistant.

```
Goal:
Audience / persona:
Constraints:
References (links, tickets):
```
